User guide

RK Excel Report Compare

Install the Excel add-in, compare an old report with a new report, and create a clear difference sheet without marking up your original worksheets.

What This Plugin Does

RK Excel Report Compare adds an RK Compare tab to Microsoft Excel. The add-in compares two open workbooks and writes the result to a separate worksheet named RK Compare in the selected new workbook.

Old report The baseline workbook and worksheet you want to compare from.
New report The workbook and worksheet that should receive the generated comparison sheet.
RK Compare The generated result worksheet with statuses, colors, and difference details.

Before You Install

  • Use Microsoft Excel desktop on Windows.
  • Use RKExcelReportCompare64-packed.xll for 64-bit Excel.
  • Use RKExcelReportCompare-packed.xll only if your Excel installation is 32-bit.
  • Most modern Microsoft Office installations are 64-bit.

Install the Add-in

  1. Download or build the packed add-in file.
  2. If the file came from the internet, right-click the .xll file in File Explorer, choose Properties, select Unblock if shown, and click OK.
  3. Open Excel.
  4. Go to File -> Options -> Add-ins.
  5. At the bottom of the window, set Manage to Excel Add-ins.
  6. Click Go.
  7. Click Browse and select the packed .xll file.
  8. Confirm the add-in is checked in the add-ins list.
  9. Close the dialog. Excel should show a new ribbon tab named RK Compare.

If Excel asks whether to enable the add-in, allow it. The plugin must be enabled before the RK Compare tab can appear.

Build the Add-in From Source

Developers can build the packed add-in from this repository folder with:

dotnet restore RKExcelReportCompare.csproj
dotnet build RKExcelReportCompare.csproj -c Release /p:Platform=x64

The packed output is created under:

bin\x64\Release\net48\publish\

Compare Two Reports

  1. Open the old report workbook in Excel.
  2. Open the new report workbook in Excel.
  3. Click RK Compare -> Compare Reports.
  4. In 1. Old report, choose the old workbook and worksheet.
  5. In 2. New report, choose the new workbook and worksheet.
  6. Choose the comparison method.
  7. If you use key matching, check the header row number and choose the old and new key headers.
  8. Click Create Comparison Sheet.

The plugin creates or replaces the RK Compare worksheet in the selected new workbook.

Choose the Right Comparison Method

Match rows by keys (recommended)

Use this mode when each report row has a stable unique identifier, such as Tag, equipment number, line number, or another item ID. The add-in matches old and new rows by that key, then compares shared headers column by column.

Compare cells by position

Use this mode when both sheets have the same layout and you want to compare cell A1 with cell A1, B1 with B1, and so on. This is useful for simple spreadsheets or exports without reliable row keys.

Understand the Result Sheet

The generated RK Compare sheet uses statuses and colors to make review faster.

Added A row or cell exists in the new report but not in the old report.
Changed A matched row or cell exists in both reports, but the value changed.
Removed A row or cell existed in the old report but no longer exists in the new report.

Key matching output

The add-in copies the selected new report sheet, inserts a Status column, and marks differences directly in the copied report. Changed cells show values in this format:

old value --> new value

A Difference details section is written below the copied report with exact old and new values. Removed rows are listed there because they are no longer present in the new report layout.

Cell position output

The add-in creates a compact difference table. The table includes these columns:

Column Meaning
Status Changed, Added, or Removed.
Key The matched key value when key matching is used.
Sheet The compared worksheet names.
Row The changed row number.
Column The changed Excel column letter.
Header The changed header name when available.
Old Value The value from the old report.
New Value The value from the new report.

Good Report Setup

  • Save or back up important workbooks before running any comparison.
  • Use clear, non-empty headers in the selected header row.
  • For key matching, choose a key column where each item appears only once.
  • Open both reports in the same Excel session before clicking Compare Reports.
  • If you open another workbook after the dialog is already open, click Refresh Workbooks.

Troubleshooting

Problem What to Try
The RK Compare tab does not appear. Close and reopen Excel, then confirm the add-in is checked under File -> Options -> Add-ins.
Excel blocks the add-in. Right-click the .xll, open Properties, and click Unblock if shown.
The workbook list is missing a report. Open both reports in Excel, then click Refresh Workbooks in the compare dialog.
No key headers are shown. Check that the selected worksheet is correct and that Header row points to the row containing column names.
The result appears in the wrong workbook. Run the comparison again and make sure the correct workbook is selected under 2. New report.
